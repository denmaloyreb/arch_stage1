# 5.6 Практическая работа

## Задача

**Что нужно сделать**

>В результате выполнения предыдущего домашнего задания (модуль 4, урок 3) у вас должны быть следующие артефакты: 
>
>* Бизнес-цели (БЦ).
>* Требования пользователей.
>* Функциональные требования (ФТ).
>* Возможные ограничения.
>* Зависимости от других систем.
>* Основные домены.
>  
>На основании этой информации:
>
>1. Составьте список на 20–30 нефункциональных требований. Для каждого домена должно быть не менее четырёх НФТ. Как минимум одно (из обязательных четырёх) должно относиться >к требованиям роста или реакции на стресс-события для системы.
>2. Определите основные атрибуты качества для описанных НФТ. Обычно 1-2 атрибута.
>3. Составьте 2–4 сценария критических бизнес-процессов. Такие сценарии охватывают более одного домена, и если они не выполняются, считается, что вся система не работает.

## Решение задачи

*По п. 1-2:*

|№ п/п|Домен|**НФТ (NFR)**|Основные атрибуты качества для НФТ|
|-----|-----|-------------|----------------------------------|
|1|Заказы|Новый заказ должен быть обработан в течение X минут|Быстродействие|
|2|Заказы|Если заказ не оплачен в течение X дней, он должен быть отменен (пользователю отправляется уведомление)|Стоимость и прибыль (COST — коммерческий атрибут)|
|3|Заказы|Для частных лиц нужно реализовать возможность приема платежей банковскими картами (с поддержкой 3D Secure), переводом, по QR-коду|Практичность, Доступность|
|4|Заказы|Для организаций создать страницу для ввода реквизитов и с кнопкой заказа обратного звонка, которая будет активной в рабочее время|Доступность, Локализация|
|5|Заказы|Запрос на резервирование товара должен быть отправлен в складскую систему в течение x мс после подтверждения заказа пользователем|Быстродействие, *Реакция на стресс-события*|
|6|Сайт|Загрузка страниц сайта (включая время отклика API и рендеринг) не должна превышать X секунд на мобильных устройствах (при минимальной скорости интернета X кбит/сек) и Y секунд на десктопах (при минимальной скорости интернета Y кбит/сек)|Быстродействие, Юзабилити|
|7|Сайт|Хранение персональных данных данных должно соответствовать требованиям 152-ФЗ|Безопасность, Локализация|
|8|Сайт|Сервер (хостинг) должен выдерживать не менее X одновременных обращений/запросов|Надежность, Доступность, *Реакция на стресс-события*|
|9|Сайт|Найти нужный товар в каталоге пользователь должен не более чем за X кликов|Юзабилити|
|10|Сайт|Сайт должен поддерживаться в браузерах согласно списку (предоставить перечень браузеров и версий)|Юзабилити, Доступность|
|11|Сайт|Для личного кабинета предусмотреть 2FA с  доставкой одноразовых паролей по SMS|Безопасность, Надежность|
|12|Складская система|База данных должна поддерживать репликацию, шардирование, полнотекстовый поиск|Быстродействие, Надежность, *Реакция на стресс-события*|
|13|Складская система|Данные о наличии товара в БД для сайте должны добавляться не позднее чем через X секунд с момента внесения изменений в систему складского учета. На это время должна осуществляться Блокировка БД на модификацию|Эксплуатационная пригодность|
|14|Складская система|Реализовать автоматическое создание заявок на закупку материалов/комплектующих при размере остатков согласно приложенной спецификации (разработать спецификацию на основе опросов разных стейкхолдеров из разных отделов)|Надежность, эксплуатационная пригодность|
|15|Складская система|Предусмотреть совместимость с системами финансового учета от *Название вендора*, т.к. они уже используются в компании|Применимость|
|16|Складская система|Доступность БД складской системы должна составлять не менее 98% времени|Надежность, Безопаность,*Реакция на стресс-события*|
|17|Логистика|Изменить адрес и время доставки может только администратор вручную, до момента изменения статуса заказа "Передан в доставку"|Стоимость и прибыль, *Реакция на стресс-события*|
|18|Логистика|Должна быть синхронизация в режиме реального времени со службами доставки (логистическими операторами) для расчета стоимости доставки онлайн|Стоимость и прибыль, Юзабилити|
|19|Логистика|Данные о клиентах (контакты, адреса доставки) должны храниться в зашифрованном виде. Обмен сведениями со службами доставки — только по VPN|Безопасность|
|20|Логистика|Реализовать интеграцию со службами доставки: "XXXX", "YYYYY", "ZZZZZ"|Доступность, Локализация|
